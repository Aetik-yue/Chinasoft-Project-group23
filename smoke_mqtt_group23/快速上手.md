# 智慧烟感 · 快速上手（环境搭建与运行）

> 本指南面向「拿到代码后从零搭环境、把项目完整跑通」的同学。
> 代码层面的技术参考（引脚表、JSON 契约、报警逻辑、文件结构）已在同目录 [README.md](README.md) 写过，本文**不再重复**，需要时点链接过去看。

---

## 一、项目简介

智慧烟感系统由三部分组成：

| 部分 | 说明 | 归谁 |
|------|------|------|
| **固件** | BearPi-HM Nano 读 MQ2 烟雾浓度，经 MQTT 上报、接收后端控制 | **本仓库** |
| **MQTT broker** | 老师提供 `47.108.58.107:1883` | 老师 |
| **后端** | 订阅上报数据、下发控制指令（Java/SpringBoot） | 组内后端同学 |

本仓库只负责：把固件烧到开发板 → 连 WiFi → 连 broker → 开始上报。后端那套不在本仓库范围，联调时让后端同学订阅 `group23` 即可。

## 二、你需要准备的东西

- 一台 Windows 电脑
- BearPi-HM Nano 开发板 + E53_SF1 扩展板（MQ2 + 蜂鸣器）+ Type-C 数据线
- 一个 **2.4GHz** 的 WiFi（Hi3861 不支持 5GHz；用手机热点要在设置里切到 2.4GHz）

## 三、环境准备

照官方《BearPi-HM Nano 十分钟上手》搭好开发环境。下面是要装的工具 + 关键步骤。

### 3.1 下载安装以下工具

| 工具 | 用途 | 下载 |
|------|------|------|
| VMware Workstation | 跑 Ubuntu 编译机 | vmware.com |
| Ubuntu 18.04 OVF 镜像 | 官方打包好的编译环境（免自己装工具链） | 百度云，见官方教程 |
| MobaXterm | SSH 连虚拟机 + 看串口日志 | mobatek.net |
| RaiDrive | 把虚拟机文件映射成 Windows 盘 | raidrive.com |
| CH340 驱动 | 开发板 USB 转串口 | wch.cn |
| HiBurn | 烧录工具 | 百度云，见官方教程 |
| VS Code | 看代码（可选） | code.visualstudio.com |

### 3.2 部署 Ubuntu 虚拟机

1. 解压 Ubuntu OVF 镜像，VMware 里「打开」该文件夹 →「导入」（磁盘预留 ≥10G）。
2. 开机，登录：账号 `bearpi` / 密码 `bearpi`。
3. 桌面右键开终端，`ifconfig`，记下除 `lo` 外那个网卡的 IP。
4. 连不上网（校园网/拨号）？VMware → 虚拟机设置 → 网络适配器改成 `NAT 模式`。

### 3.3 Windows 远程连虚拟机

MobaXterm → `Session` → `SSH` → 填上面记的 IP → 账号 `bearpi` / 密码 `bearpi`。

### 3.4 把虚拟机文件映射到 Windows

RaiDrive → 添加 → `SFTP`，填虚拟机 IP，账号密码都是 `bearpi`，**取消勾选「只读」**。
映射后 Windows 会多出一个盘，对应虚拟机 `/home/bearpi`。

> 这一步是为了让 Windows 上的 HiBurn / VS Code 能直接访问虚拟机里编译出的固件。后面编译出的 `.bin` 就在这个映射盘里。

## 四、获取源码并集成本样例

本仓库只含 `smoke_mqtt_group23/` 一个样例，得先拿到官方 BearPi 完整工程，再把本样例放进去。

### 4.1 clone 官方 BearPi 工程

MobaXterm 里：

```bash
cd /home/bearpi
mkdir project && cd project
git clone https://gitee.com/bearpi/bearpi-hm_nano.git -b master
```

### 4.2 把本样例放进工程

把本仓库的 `smoke_mqtt_group23/` 整个目录拷到官方工程的：

```
bearpi-hm_nano/applications/BearPi/BearPi-HM_Nano/sample/smoke_mqtt_group23/
```

（用 RaiDrive 映射盘直接拖进去即可。）

### 4.3 在 BUILD.gn 里启用本样例

编辑 `applications/BearPi/BearPi-HM_Nano/sample/BUILD.gn`，让 `features` 列表里**只有本样例一行没注释**，其余全部保持注释：

```gn
lite_component("app") {
    features = [
        "smoke_mqtt_group23:smoke_mqtt_group23",
        # 其它样例全部保持注释
    ]
}
```

> 同一时刻只能启用一个样例，否则会编译冲突。

## 五、配置固件参数

编辑 `smoke_mqtt_group23/iot_smoke.c` 顶部的「用户配置区」，至少改这几项：

```c
#define WIFI_SSID   "你的WiFi名"      // 必须 2.4GHz
#define WIFI_PWD    "你的WiFi密码"
#define BROKER_IP   "47.108.58.107"   // 老师给的 broker
#define BROKER_PORT 1883
#define PUB_TOPIC   "group23"        // 上报主题，后端订阅这个
#define CLIENT_ID   "bearpi_group23" // 客户端ID，别和别人重复
```

> 各配置项完整说明见 [README.md](README.md) 的「WiFi 配置」「MQTT 配置」两节。
>
> ⚠️ **坑**：broker 地址/端口/是否需要账号密码，以老师最终确认为准。目前固件**没设** MQTT 用户名密码，若 broker 要认证会连不上，找后端同学对齐。

## 六、（必做）调大 paho MQTT 收包线程栈

**这一步不做，板子一收 MQTT 消息就崩。** 现象是串口打印 `[ERR] CURRENT task MQTTTask stack overflow!` 然后死机。

原因：本样例在 MQTT 下行回调里用了 `cJSON_Parse` + `printf`，而 paho 库默认给收包线程 `MQTTTask` 只配了 2KB 栈，不够。

改官方工程里的这个文件：

```
third_party/paho_mqtt/MQTTClient-C/src/liteOS/MQTTLiteOS.c
```

找到 `ThreadStart()` 函数里的：

```c
attr.stack_size = 2048;
```

改成：

```c
attr.stack_size = 4096;
```

> 注意这是改 `third_party/`，不在本样例目录里，所以 clone 官方工程后**必须手动改这一处**，否则队友烧上去必崩。

## 七、编译

MobaXterm 里进工程根目录编译：

```bash
cd /home/bearpi/project/bearpi-hm_nano
python build.py BearPi-HM_Nano
```

等到出现 `BUILD SUCCESS` 即成功。固件文件在（RaiDrive 映射盘里可直接看到）：

```
bearpi-hm_nano/out/ 下找到 Hi3861_wifiiot_app_allinone.bin
```

## 八、烧录

1. Type-C 连接开发板与电脑，装好 CH340 驱动。
2. **关闭 VMware 的 USB 捕获**（否则 Windows 看不到串口）：VMware → 虚拟机 → 可移动设备 → 把 CH340 断开给虚拟机；不行就直接挂起虚拟机再插拔 USB。
3. Windows 设备管理器确认开发板的 COM 号。
4. 打开 HiBurn：
   - `Refresh` → `COM` 选上面的号。
   - `Setting` → `Com settings` → `Baud` 改 `921600`，确定。
   - `Select file` → 选 `Hi3861_wifiiot_app_allinone.bin`。
   - 勾 `Auto burn` → 点 `Connect`（按钮变成 `Disconnect`）。
5. 按开发板 `RESET` 键，开始烧录。
6. 出现 `Execution Successful` → 烧完，点 `Disconnect`（释放串口给 MobaXterm）。

## 九、看串口日志

MobaXterm → `Session` → `Serial` → 选同一个 COM → `Speed` 填 `115200` → OK。复位开发板，应看到类似：

```
[MQTT] connecting to 47.108.58.107:1883 ...
[MQTT] connected!
[MQTT] background receive task started
[MQTT] subscribed to 'group23-s-to-h'
[SMOKE] sensor calibrated, start reporting...
[SMOKE] ppm: 87.123
[MQTT] publish to 'group23': {"ppm":87.123}
```

> 串口报 `Unable to open`：HiBurn 是不是还连着没 Disconnect（第八-6 步）。

## 十、连接后端联调

固件上电后会自动：连 WiFi → 连 broker `47.108.58.107:1883` → 每秒往 `group23` 发 `{"ppm":N}`。

- **后端同学**：订阅 `group23` 收 ppm；要控制板子，往 `group23-s-to-h` 发 `{"sensor":0/1}` 或 `{"threshold":N}`。
- 主题与 JSON 契约详见 [README.md](README.md) 的「后端控制」一节。

后端那套（Java/SpringBoot）由组内后端同学负责，不在本仓库。

## 十一、端到端验证清单

按顺序确认：

- [ ] 串口出现 `[MQTT] connected!` 与 `subscribed to 'group23-s-to-h'`
- [ ] 串口每秒打印 `[SMOKE] ppm:` 与 `[MQTT] publish to 'group23'`
- [ ] 后端能收到 `group23` 的 `{"ppm":N}`
- [ ] 后端发 `{"sensor":0}` → 板子停止打印 ppm/publish（串口出现 `[CTRL] sensor OFF`），但 MQTT 仍连着
- [ ] 后端发 `{"sensor":1}` → 恢复上报
- [ ] 后端发 `{"threshold":200}` → 串口出现 `[CTRL] threshold -> 200 ppm`，之后报警按 200 判断
- [ ] 对着 MQ2 吹烟 → ppm 上升超过阈值 → 蜂鸣器响 + LED 快闪

## 十二、常见问题

| 现象 | 原因 / 解决 |
|------|-------------|
| WiFi 连不上 | 不是 2.4GHz，或密码错。手机热点要在设置里切 2.4GHz |
| `[MQTT] connect failed` 反复重连 | broker 地址/端口错，或 broker 要账号密码而固件没设 → 找后端确认是否需认证 |
| `MQTTTask stack overflow!` | 没做第六步的栈修改，或改了之后没重新编译；还溢出就把 `attr.stack_size` 再调大（如 6144） |
| HiBurn 烧不进 | VMware 抢了 USB（关捕获）/ COM 被占（关 MobaXterm 串口）/ 没按 RESET |
| 串口 `Unable to open` | HiBurn 还连着没 Disconnect |
| ppm 数值离谱 | 开机校准那 1 秒环境不干净（要洁净空气），或 MQ2 没预热几分钟 |
